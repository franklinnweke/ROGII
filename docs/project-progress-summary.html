Project Summary

ROGII Wellbore Geology Prediction

This project is Franklin Nweke's Kaggle competition work for predicting true vertical thickness in horizontal wellbore geology data. The work is being framed as a serious applied ML and petroleum engineering portfolio project, not as an overclaimed result.

What We Have Done So Far

  1. Created a dedicated project repository. The work is under its own standalone GitHub repo, franklinnweke/ROGII, separate from other projects.
  2. Downloaded and organized the Kaggle competition data locally. Raw Kaggle data lives in data/raw/, while generated outputs, submissions, and working files are ignored by Git so private/large competition artifacts do not get pushed.
  3. Built the first project structure. The repo now has Python source code, scripts, tests, docs, notebooks, and a clean ignore policy.
  4. Added data audit and baseline tooling. Earlier commits added scripts to inspect the dataset, validate baseline behavior, and generate a Kaggle submission file.
  5. Researched the competition direction. We reviewed Kaggle notebooks and domain references. The key finding is that this problem is not best approached first as a generic tabular regression task. The public evidence points toward continuity baselines, gamma-ray/typewell correlation, and guarded blends.
  6. Protected authorship hygiene. The repo content was scanned for unwanted authorship labels, and none were found in the pushed project files.

What The Last Push Achieved

The latest GitHub update changed the project direction from an interpolation-first baseline to a stronger, competition-aligned flat-tail baseline.

Default Baseline Corrected

Prediction now carries the last known TVT_input value through the hidden tail, instead of extrapolating an increasing/decreasing line.

Submission Script Updated

scripts/make_baseline_submission.py now supports --strategy flat and uses it by default.

Validation Made More Honest

Validation now targets the original missing TVT_input tails in training wells by default, matching the actual Kaggle submission setup more closely.

Kaggle Notebook Draft Added

Added notebooks/rogii_flat_tail_baseline.py, a simple Kaggle-ready script that writes submission.csv.
